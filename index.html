1. Components of a Computer System

Memory

Memory stores data and instructions. Types include RAM (volatile) for temporary storage, and ROM (non-volatile) for firmware.

Processor (CPU)

The brain of the computer, executing instructions using the Arithmetic Logic Unit (ALU) and Control Unit.

I/O Devices

Input devices (keyboard, mouse) and output devices (monitor, printer) facilitate communication between user and computer.

Storage

Permanent data storage devices like HDD (magnetic) and SSD (flash memory).

Operating System

Manages hardware/software resources, user interface, file and memory management.

Assembler, Compiler, Interpreter, Loader & Linker

  • Assembler: Converts assembly language to machine code.
  • Compiler: Translates high-level code (e.g. C) to machine code.
  • Interpreter: Executes code line-by-line.
  • Loader: Loads executable code into memory.
  • Linker: Combines object files into a single executable.

Algorithm Concepts

An algorithm is a step-by-step problem-solving process represented by:

  • Flowchart: Graphical diagram of logic flow.
  • Pseudo Code: Plain language representation close to code syntax.

Programming involves converting algorithms into source code.

Tip: Understand algorithm flow with flowcharts and pseudo code before coding!

2. Programming Basics in C

Structure of a C Program

#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}

Writing and Executing

Write code in a .c file, compile with gcc, then run the executable.

Errors

  • Syntax Errors: Violations of language rules.
  • Logical Errors: Code runs but produces wrong output.

Object & Executable Code

Compiler produces object code, which the linker combines to create executable code.

Components of C Language

  • Keywords (reserved words)
  • Data Types (int, float, char)
  • Operators (+, -, *, etc.)

Standard I/O

printf() outputs; scanf() receives input.

Variables & Storage Classes

  • auto: Default local variable
  • static: Retains value between calls
  • extern: Declares global variables
  • register: Stores in CPU register (fast access)
Tip: Practice compiling and running simple C programs to get familiar!

3. Arithmetic Expressions and Operators

Operators let you compute, compare, and manipulate data.

  • Arithmetic: +, -, *, /, %
  • Relational: ==, !=, >, <, >=, <=
  • Logical: &&, ||, !
  • Bitwise: &, |, ^, ~, <<, >>
  • Assignment: =, +=, -=, etc.

Operator Precedence & Associativity

Defines order of evaluation; e.g., * and / before + and -, left-to-right associativity.

Tip: Operator precedence errors are common; use parentheses to be explicit!

4. Conditional Branching

if Statement

if (condition) {
    // code
}

if-else Statement

if (condition) {
    // true block
} else {
    // false block
}

switch Statement

switch (variable) {
    case value1:
        // code
        break;
    default:
        // default code
}
Tip: Use switch for multiple exclusive cases for clarity & efficiency.

5. Iteration and Loops

while Loop

while (condition) {
    // code
}

do-while Loop

do {
    // code
} while (condition);

for Loop

for(init; condition; increment) {
    // code
}

Control Statements

  • break: exits loop
  • continue: skips to next iteration
  • goto: jumps to label (use sparingly)
Tip: Avoid goto; use structured loops and functions for cleaner code.

7. Searching and Sorting Algorithms

Linear Search

int linearSearch(int arr[], int n, int key) {
    for(int i=0; i < n; i++) {
        if(arr[i] == key) return i;
    }
    return -1;
}

Bubble Sort

void bubbleSort(int arr[], int n) {
    for(int i = 0; i < n - 1; i++) {
        for(int j = 0; j < n - i - 1; j++) {
            if(arr[j] > arr[j+1]) {
                int temp = arr[j];
                arr[j] = arr[j+1];
                arr[j+1] = temp;
            }
        }
    }
}
Tip: Understand basic algorithms thoroughly before moving to complex ones.

8. Functions

Function Declaration and Definition

void functionName(parameters) {
    // body
}

Passing Arrays to Functions

void printArray(int arr[], int size) {
    for(int i=0; i < size; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

Call by Value vs Call by Reference

void callByValue(int x) {
    x = 20; // original unaffected
}

void callByReference(int *x) {
    *x = 20; // original changed
}

Recursive Functions

int factorial(int n) {
    if(n == 0) return 1;
    else return n * factorial(n-1);
}
Tip: Recursion needs a base case; otherwise, it leads to infinite calls!

9. Pointers

Pointer Declaration

int *ptr; // pointer to int

Passing Pointers to Functions

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

Pointer Operations

  • Dereferencing: Access value using *ptr.
  • Pointer arithmetic: Increment pointer to access array elements.

Pointer Arrays

char *names[3] = { "Alice", "Bob", "Charlie" };

Structures and Pointers

struct Student {
    char name[50];
    int age;
};

void printStudent(struct Student *s) {
    printf("Name: %s, Age: %d\n", s->name, s->age);
}
Tip: Use pointers carefully and always initialize them!
